## NOTICE file contents

The `NOTICE` file at the root of the NuttX source tree reads:

```
Apache NuttX
Copyright 2020 The Apache Software Foundation

This product includes software developed at
The Apache Software Foundation (http://www.apache.org/).
```

Beyond that standard ASF notice, the NOTICE file documents two additional items you should be aware of when building NuttX for a product:

### FAT long filename patent risk

Enabling `CONFIG_FAT_LFN` in your NuttX configuration activates FAT long filename support. The NOTICE file states:

> If `CONFIG_FAT_LFN` is defined in your NuttX configuration file, then there may be some legal, patent issues.

Microsoft holds several US patents covering FAT long filename implementations:

| Patent | Subject |
|---|---|
| US 5,745,902 | Method and system for accessing a file using file names having different file name formats |
| US 5,579,517 | Common name space for long and short filenames |
| US 5,758,352 | Common name space for long and short filenames |
| US 6,286,013 | Method and system for providing a common name space for long and short file names in an OS |

<Warning>
  The FAT long filename feature (`CONFIG_FAT_LFN`) may require a patent licence from Microsoft if used in a commercial product. Review the NOTICE file and consult legal counsel before enabling this feature. Short filename (8.3) FAT use is generally considered unaffected by these patents.
</Warning>

### NXP SPIFI restriction

The NOTICE file also records a use restriction on SPIFI (Serial Peripheral Interface Flash Interface) code for the LPC43xx:

> Code supporting SPIFI for the LCP43xx has restricted usage by NXP. Permission to use, copy, modify, and distribute this software and its documentation is hereby granted, under NXP Semiconductors' and its licensor's relevant copyrights in the software, without fee, **provided that it is used in conjunction with NXP Semiconductors microcontrollers**.

The affected files are:
- `arch/arm/src/lpc43xx/hardware/lpc43_spifi.h`
- `arch/arm/src/lpc43xx/spifi/inc/spifilib_api.h`
- `arch/arm/src/lpc43xx/spifi/inc/spifilib_dev.h`
- `arch/arm/src/lpc43xx/spifi/inc/private/spifilib_chiphw.h`
- `arch/arm/src/lpc43xx/spifi/src/spifilib_dev_common.c`
- `arch/arm/src/lpc43xx/spifi/src/spifilib_fam_standard_cmd.c`

> "This product includes software developed by the University of California, Berkeley and its contributors."

## Contributing back

PX4's NuttX fork follows the Apache NuttX contribution process for changes that are not PX4-specific. The `CONTRIBUTING.md` file in the repository root describes the requirements:

- **Commit messages** must be descriptive. Use a `<keyword>: <message>` prefix to give context (e.g., `sched: fix compiler warning`). A body paragraph is encouraged for non-trivial changes.
- **Pull request template** — fill in the `Summary`, `Impact`, and `Testing` sections of the PR template. Do not remove these sections.
- **Coding standard** — all modified files must conform to the [NuttX C Coding Standard](https://nuttx.apache.org/docs/latest/contributing/coding_style.html). The `lint.yml` CI workflow automatically checks compliance using the `nxstyle` tool. You must fix style errors in files you modify even if you did not introduce them.
- **CI** — all five CI workflows (build, check, lint, doc, docker) must pass before a PR can be merged.

<Info>
  Changes that improve NuttX for general embedded use cases — not specific to PX4 — should be submitted to the upstream Apache NuttX project at [github.com/apache/incubator-nuttx](https://github.com/apache/incubator-nuttx) rather than only to the PX4 fork.
</Info>
