NuttX provides four primary inter-process communication (IPC) mechanisms: POSIX counting semaphores, POSIX message queues, signals, and pipes. Semaphores and message queues live in `sched/semaphore/` and `sched/mqueue/` respectively; their public interfaces are declared in `include/semaphore.h` and `include/mqueue.h`. Signals are described separately in the [POSIX API reference](/reference/posix-api).

## POSIX semaphores

Semaphores are the primary synchronization primitive in NuttX. Every mutex, condition variable, and message queue is ultimately backed by a `sem_t`. NuttX supports priority inheritance on semaphores when `CONFIG_PRIORITY_INHERITANCE=y`.

### sem_t structure

```c
/* From include/semaphore.h */

struct sem_s
{
  volatile int16_t semcount;  /* >0: available count; <0: tasks waiting */

#ifdef CONFIG_PRIORITY_INHERITANCE
  uint8_t flags;              /* PRIOINHERIT_FLAGS_DISABLE if inheritance is off */
  struct semholder_s holder[2]; /* Tracks which tasks hold the semaphore */
#endif
};

typedef struct sem_s sem_t;

/* Protocol attribute values */
#define SEM_PRIO_NONE     0  /* No priority inheritance */
#define SEM_PRIO_INHERIT  1  /* Priority inheritance enabled */
#define SEM_PRIO_PROTECT  2  /* Priority ceiling protocol */

/* Static initializer */
#define SEM_INITIALIZER(c) {(c)}  /* flat: initializes semcount to c */
```

### Semaphore function reference

<AccordionGroup>
  <Accordion title="sem_init() and sem_destroy()">
    ```c
    int sem_init(FAR sem_t *sem, int pshared, unsigned int value);
    int sem_destroy(FAR sem_t *sem);
    ```

    <ParamField path="sem" type="FAR sem_t *" required>
      Pointer to the semaphore object to initialize. The caller is responsible for storage (stack, static, or heap).
    </ParamField>

    <ParamField path="pshared" type="int" required>
      Set to `0` for a semaphore shared between threads of the same task group. NuttX currently ignores non-zero values; named semaphores (`sem_open`) are the supported mechanism for inter-process sharing.
    </ParamField>

    <ParamField path="value" type="unsigned int" required>
      Initial count. A value of `0` means the semaphore is unavailable; the first `sem_wait()` will block. A value of `1` makes a binary semaphore (mutex-like).
    </ParamField>

    <ResponseField name="return" type="int">
      Returns `OK` (0) on success, or `-1` with `errno` set on failure (`EINVAL`, `ENOSPC`).
    </ResponseField>
  </Accordion>

  <Accordion title="sem_wait(), sem_trywait(), sem_timedwait()">
    ```c
    int sem_wait(FAR sem_t *sem);
    int sem_trywait(FAR sem_t *sem);
    int sem_timedwait(FAR sem_t *sem,
                      FAR const struct timespec *abstime);
    int sem_clockwait(FAR sem_t *sem, clockid_t clockid,
                      FAR const struct timespec *abstime);
    ```

    `sem_wait()` decrements the semaphore count. If the count is already zero, the calling task blocks until another task calls `sem_post()`.

    `sem_trywait()` returns immediately with `errno = EAGAIN` if the count is zero.

    `sem_timedwait()` blocks until the semaphore is posted or the absolute deadline `abstime` (on `CLOCK_REALTIME`) passes. Returns `-1` with `errno = ETIMEDOUT` on deadline expiry.

    `sem_clockwait()` is a NuttX extension that accepts an explicit clock ID (`CLOCK_REALTIME` or `CLOCK_MONOTONIC`).

    <ParamField path="sem" type="FAR sem_t *" required>
      The semaphore to wait on.
    </ParamField>

    <ParamField path="abstime" type="FAR const struct timespec *" required>
      Absolute expiry time (not a relative timeout). Compute with `clock_gettime()` plus an offset.
    </ParamField>

    <ResponseField name="return" type="int">
      Returns `OK` (0) on success. On failure, returns `-1` with `errno` set to `EINVAL`, `EINTR`, `ETIMEDOUT`, or `EAGAIN`.
    </ResponseField>
  </Accordion>

  <Accordion title="sem_post() and sem_getvalue()">
    ```c
    int sem_post(FAR sem_t *sem);
    int sem_getvalue(FAR sem_t *sem, FAR int *sval);
    ```

    `sem_post()` increments the semaphore count and unblocks the highest-priority waiting task, if any.

    `sem_getvalue()` reads the current count into `*sval`. A negative value indicates the number of tasks waiting.

    <Warning>
      `sem_post()` is safe to call from interrupt context. `sem_wait()` is not — calling it from an ISR will deadlock the system.
    </Warning>
  </Accordion>

  <Accordion title="Named semaphores">
    Named semaphores are available when `CONFIG_FS_NAMED_SEMAPHORES=y`. They allow unrelated processes to share a semaphore via a filesystem path.

    ```c
    FAR sem_t *sem_open(FAR const char *name, int oflag, ...);
    int        sem_close(FAR sem_t *sem);
    int        sem_unlink(FAR const char *name);
    ```

    Typical flags for `oflag`: `O_CREAT | O_EXCL`. The variadic arguments are `mode_t mode, unsigned int value` when `O_CREAT` is set.
  </Accordion>

  <Accordion title="sem_setprotocol() — priority inheritance control">
    A NuttX extension that controls the priority inheritance protocol on a per-semaphore basis.

    ```c
    int sem_setprotocol(FAR sem_t *sem, int protocol);
    int sem_getprotocol(FAR sem_t *sem, FAR int *protocol);
    ```

    <ParamField path="protocol" type="int" required>
      One of `SEM_PRIO_NONE`, `SEM_PRIO_INHERIT`, or `SEM_PRIO_PROTECT`.
    </ParamField>

    <Note>
      When a semaphore is used for one-way signaling (Task A posts, Task B waits), priority inheritance can cause unexpected priority boosts on Task A. Call `sem_setprotocol(sem, SEM_PRIO_NONE)` immediately after `sem_init()` to disable inheritance for such semaphores.
    </Note>
  </Accordion>
</AccordionGroup>

### Semaphore usage example

<CodeGroup>
```c binary semaphore (mutex replacement)
#include <semaphore.h>
#include <stdio.h>

static sem_t g_lock;

void shared_resource_init(void)
{
  sem_init(&g_lock, 0, 1);          /* Binary: starts unlocked */
  sem_setprotocol(&g_lock, SEM_PRIO_INHERIT);
}

void shared_resource_acquire(void)
{
  sem_wait(&g_lock);
}

void shared_resource_release(void)
{
  sem_post(&g_lock);
}
```
```c timed wait example
#include <semaphore.h>
#include <time.h>
#include <errno.h>

int wait_with_timeout(FAR sem_t *sem, long timeout_ms)
{
  struct timespec ts;

  clock_gettime(CLOCK_REALTIME, &ts);
  ts.tv_sec  += timeout_ms / 1000;
  ts.tv_nsec += (timeout_ms % 1000) * 1000000L;
  if (ts.tv_nsec >= 1000000000L)
    {
      ts.tv_sec++;
      ts.tv_nsec -= 1000000000L;
    }

  if (sem_timedwait(sem, &ts) != OK)
    {
      if (errno == ETIMEDOUT)
        return -ETIMEDOUT;
      return -errno;
    }

  return OK;
}
```
</CodeGroup>

## POSIX message queues

Message queues provide buffered, prioritized message passing between tasks. They are declared in `include/mqueue.h` and implemented in `sched/mqueue/`.

### mq_attr and mqd_t

```c
/* From include/mqueue.h */

struct mq_attr
{
  size_t   mq_maxmsg;   /* Maximum number of messages in queue */
  size_t   mq_msgsize;  /* Maximum size of each message in bytes */
  unsigned mq_flags;    /* Queue flags (0 or MQ_NONBLOCK) */
  size_t   mq_curmsgs;  /* Number of messages currently in queue */
};

typedef int mqd_t;        /* Message queue descriptor */

#define MQ_NONBLOCK O_NONBLOCK  /* Non-blocking flag for mq_attr.mq_flags */
```

### Message queue function reference

<AccordionGroup>
  <Accordion title="mq_open() and mq_close()">
    ```c
    mqd_t mq_open(FAR const char *mq_name, int oflags, ...);
    int   mq_close(mqd_t mqdes);
    int   mq_unlink(FAR const char *mq_name);
    ```

    <ParamField path="mq_name" type="const char *" required>
      Name of the message queue. By POSIX convention, names start with `/`. Example: `"/sensor_data"`.
    </ParamField>

    <ParamField path="oflags" type="int" required>
      Open flags. Common combinations: `O_RDONLY`, `O_WRONLY`, `O_RDWR`, optionally `| O_CREAT | O_EXCL`. Add `O_NONBLOCK` to enable non-blocking mode.
    </ParamField>

    <ParamField path="..." type="mode_t mode, struct mq_attr *attr">
      Required when `O_CREAT` is set. `mode` is the permission bits (often `0666`). `attr` specifies `mq_maxmsg` and `mq_msgsize`; pass `NULL` for defaults.
    </ParamField>

    <ResponseField name="return (mq_open)" type="mqd_t">
      Returns a valid descriptor on success, or `(mqd_t)-1` with `errno` set on failure (`ENOENT`, `EACCES`, `EMFILE`).
    </ResponseField>

    `mq_close()` releases the descriptor. The queue persists until `mq_unlink()` is called and all descriptors are closed.
  </Accordion>

  <Accordion title="mq_send() and mq_timedsend()">
    ```c
    int mq_send(mqd_t mqdes, FAR const char *msg,
                size_t msglen, unsigned int prio);

    int mq_timedsend(mqd_t mqdes, FAR const char *msg,
                     size_t msglen, unsigned int prio,
                     FAR const struct timespec *abstime);
    ```

    <ParamField path="mqdes" type="mqd_t" required>
      Descriptor returned by `mq_open()`, opened with write access (`O_WRONLY` or `O_RDWR`).
    </ParamField>

    <ParamField path="msg" type="const char *" required>
      Pointer to the message bytes. NuttX copies the message into the queue.
    </ParamField>

    <ParamField path="msglen" type="size_t" required>
      Length of the message in bytes. Must be ≤ `mq_msgsize` set at queue creation.
    </ParamField>

    <ParamField path="prio" type="unsigned int" required>
      Message priority. Messages are delivered in descending priority order. Equal-priority messages are delivered in FIFO order.
    </ParamField>

    <ParamField path="abstime" type="FAR const struct timespec *" required>
      Absolute timeout for `mq_timedsend()`. If the queue is full and the deadline passes, returns `-1` with `errno = ETIMEDOUT`.
    </ParamField>

    <ResponseField name="return" type="int">
      Returns `OK` (0) on success or `-1` with `errno` set on failure.
    </ResponseField>
  </Accordion>

  <Accordion title="mq_receive() and mq_timedreceive()">
    ```c
    ssize_t mq_receive(mqd_t mqdes, FAR char *msg,
                       size_t msglen, FAR unsigned int *prio);

    ssize_t mq_timedreceive(mqd_t mqdes, FAR char *msg,
                            size_t msglen, FAR unsigned int *prio,
                            FAR const struct timespec *abstime);
    ```

    <ParamField path="mqdes" type="mqd_t" required>
      Descriptor opened with read access (`O_RDONLY` or `O_RDWR`).
    </ParamField>

    <ParamField path="msg" type="FAR char *" required>
      Buffer into which the received message is copied.
    </ParamField>

    <ParamField path="msglen" type="size_t" required>
      Size of `msg` buffer. Must be ≥ `mq_msgsize`; otherwise `mq_receive()` returns `-1` with `errno = EMSGSIZE`.
    </ParamField>

    <ParamField path="prio" type="FAR unsigned int *">
      If non-NULL, receives the priority of the dequeued message.
    </ParamField>

    <ResponseField name="return" type="ssize_t">
      Returns the number of bytes received on success, or `-1` with `errno` set on failure.
    </ResponseField>
  </Accordion>

  <Accordion title="mq_notify(), mq_setattr(), mq_getattr()">
    ```c
    int mq_notify(mqd_t mqdes,
                  FAR const struct sigevent *notification);

    int mq_setattr(mqd_t mqdes,
                   FAR const struct mq_attr *mq_stat,
                   FAR struct mq_attr *oldstat);

    int mq_getattr(mqd_t mqdes, FAR struct mq_attr *mq_stat);
    ```

    `mq_notify()` arranges for `notification` to be delivered when a message arrives on an empty queue. Only one process may register per queue at a time.

    `mq_setattr()` changes the `mq_flags` field (currently only `O_NONBLOCK`). The previous attributes are stored in `oldstat` if non-NULL.

    `mq_getattr()` reads current queue attributes including the live `mq_curmsgs` count.
  </Accordion>
</AccordionGroup>

### Message queue usage example

<CodeGroup>
```c sender
#include <mqueue.h>
#include <fcntl.h>
#include <stdint.h>
#include <stdio.h>

#define QUEUE_NAME "/imu_data"
#define MAX_MSG    64

typedef struct { float x; float y; float z; } vec3_t;

void send_imu_sample(float ax, float ay, float az)
{
  struct mq_attr attr;
  attr.mq_maxmsg  = 10;
  attr.mq_msgsize = sizeof(vec3_t);
  attr.mq_flags   = 0;

  mqd_t mq = mq_open(QUEUE_NAME, O_WRONLY | O_CREAT, 0666, &attr);
  if (mq == (mqd_t)-1)
    {
      perror("mq_open");
      return;
    }

  vec3_t sample = { ax, ay, az };
  mq_send(mq, (FAR const char *)&sample, sizeof(sample), 0);
  mq_close(mq);
}
```
```c receiver
#include <mqueue.h>
#include <fcntl.h>
#include <stdio.h>

#define QUEUE_NAME "/imu_data"

typedef struct { float x; float y; float z; } vec3_t;

void receive_loop(void)
{
  mqd_t mq = mq_open(QUEUE_NAME, O_RDONLY);
  if (mq == (mqd_t)-1)
    {
      perror("mq_open");
      return;
    }

  vec3_t sample;
  unsigned int prio;

  for (;;)
    {
      ssize_t n = mq_receive(mq, (FAR char *)&sample,
                             sizeof(sample), &prio);
      if (n > 0)
        {
          printf("accel: %.3f %.3f %.3f (prio %u)\n",
                 sample.x, sample.y, sample.z, prio);
        }
    }

  mq_close(mq);
}
```
</CodeGroup>

## Pipes

Pipes provide a unidirectional byte stream between a writer and a reader. They are available when `CONFIG_PIPES=y`.

```c
#include <unistd.h>

int pipe(int pipefd[2]);   /* pipefd[0] = read end, pipefd[1] = write end */
```

For named pipes (FIFOs), NuttX also supports `mkfifo()` when the filesystem is configured to allow it:

```c
#include <sys/stat.h>
int mkfifo(FAR const char *pathname, mode_t mode);
```

<Note>
Pipes are byte-stream oriented and do not preserve message boundaries. Use message queues (`mq_*`) when you need discrete, prioritized messages.
</Note>

## Choosing the right IPC primitive

| Mechanism | Buffering | Priority | IRQ-safe post | Best for |
|-----------|-----------|----------|---------------|----------|
| Semaphore | None | Inherited | `sem_post()` yes | Mutual exclusion, signaling |
| Message queue | Yes (N msgs) | Yes | No | Multi-producer/consumer pipelines |
| Signal | No | Signal priority | `kill()` yes | Asynchronous event notification |
| Pipe | Yes (byte stream) | No | No | Shell pipelines, stdio redirection |
