---
title: "STM32 board support in NuttX"
description: "NuttX supports STMicroelectronics STM32F4, STM32F7, STM32H7, and STM32L4 MCU families with boards ranging from Discovery kits to Nucleo-144 targets."
---

STMicroelectronics STM32 microcontrollers are among the most widely supported silicon families in NuttX. The Cortex-M4/M7 cores, rich peripheral sets, and low-cost development boards have made STM32 the default choice for many NuttX developers. Support spans four major sub-families: the STM32F4 (Cortex-M4, up to 180 MHz), the STM32F7 (Cortex-M7, up to 216 MHz), the STM32H7 (Cortex-M7, up to 480 MHz), and the STM32L4 (Cortex-M4 ultra-low-power).

<Note>
All STM32 chip-level drivers live under `arch/arm/src/stm32/`, `arch/arm/src/stm32f7/`, `arch/arm/src/stm32h7/`, and `arch/arm/src/stm32l4/`. Board-specific code lives under `boards/arm/stm32*/`.
</Note>

  <Tab title="STM32H7">
    The STM32H7 sub-family uses the ARM Cortex-M7 at up to 480 MHz and includes 2 MB of Flash, 1 MB of SRAM, and a hardware JPEG codec.

    **Boards in `boards/arm/stm32h7/`:**

    | Board directory | Description |
    |---|---|
    | `nucleo-h743zi` | NUCLEO-H743ZI with STM32H743ZI, 400 MHz, 2 MB Flash, 1 MB SRAM |
    | `nucleo-h743zi2` | NUCLEO-H743ZI2 (revised hardware) |
    | `stm32h747i-disco` | STM32H747I-DISCOVERY dual-core (Cortex-M7 + M4) |

    **Available configurations on `nucleo-h743zi`:**

    The STM32H7 port took longer to mature than STM32F4/F7 due to significant RCC changes between families. All listed boards are functional as of recent NuttX releases.

## Configuring and building

<Steps>
  <Step title="Select a board configuration">
    Run `configure.sh` (or `tools/configure.py`) with the `<board>:<config>` pair.

    ```bash
    cd nuttx
    ./tools/configure.sh stm32f4discovery:nsh
    ```
  </Step>
  <Step title="Optionally customise via menuconfig">
    ```bash
    make menuconfig
    ```
  </Step>
  <Step title="Build the firmware">
    ```bash
    make -j$(nproc)
    ```

    The output files are `nuttx` (ELF) and `nuttx.bin` (raw binary).
  </Step>
  <Step title="Flash the board">
    Use OpenOCD, ST-LINK, or `st-flash` to program the binary:

    ```bash
    st-flash write nuttx.bin 0x08000000
    ```
  </Step>
</Steps>
