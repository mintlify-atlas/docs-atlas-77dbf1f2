---

NuttShell (NSH) is a compact, bash-inspired interactive shell for NuttX that lives in `apps/nshlib/`. It is the primary human interface to a running NuttX system, reachable over a serial console, USB CDC/ACM connection, or Telnet session. NSH provides file system navigation, task and process management, network configuration, memory inspection, and a scripting engine—all selectable individually at build time so the memory footprint stays minimal on resource-constrained targets. When you see the `nsh>` prompt you have access to a rich set of built-in commands that exercise every major NuttX subsystem.

## Starting NSH

When NuttX boots and NSH is enabled, the welcome message appears on the configured console:

```
NuttShell (NSH) NuttX-12.x
nsh>
```

<Note>
  USB console start-up requires pressing **Enter** three times after the USB serial device enumerates. This ensures the host terminal application is ready and discards any AT modem commands the host OS sends automatically.
</Note>

Enable NSH in Kconfig:

```kconfig
CONFIG_NSH_LIBRARY=y
CONFIG_NSH_CONSOLE=y        # serial console front-end
# CONFIG_NSH_TELNET=y       # optional: Telnet front-end (requires TCP)
```

## Scripting

NSH supports `if-then[-else]-fi`, `while-do-done`, and `until-do-done` constructs, making it suitable for startup and test scripts.

```bash
#!/bin/nsh
# /etc/init.d/rcS — example startup script

# Mount a FAT RAM disk at /tmp
mkrd -m 1 -s 512 1024
mkfatfs /dev/ram1
mount -t vfat /dev/ram1 /tmp

# Bring up the network interface
ifup eth0
```

Scripts are executed with the `sh` command:

```
nsh> sh /etc/init.d/rcS
```

Variable substitution, command output capture (back-quotes), and string concatenation are available when `CONFIG_NSH_CMDPARMS` and `CONFIG_NSH_ARGCAT` are enabled.

## Startup scripts

<Steps>
  <Step title="Enable ROMFS /etc">
    Set `CONFIG_NSH_ROMFSETC=y`. NSH will mount a tiny ROMFS image at `/etc` containing `init.d/rc.sysinit` and `init.d/rcS`.
  </Step>

  <Step title="Customise rc.sysinit">
    Edit `apps/nshlib/rc.sysinit.template`. The default creates a FAT RAM disk at `/tmp`:

    ```bash
    mkrd -m 1 -s 512 1024
    mkfatfs /dev/ram1
    mount -t vfat /dev/ram1 /tmp
    ```
  </Step>

  <Step title="Rebuild the ROMFS image">
    Run `tools/mkromfsimg.sh` to regenerate `apps/nshlib/nsh_romfsimg.h` from the updated template, then rebuild NuttX.
  </Step>

  <Step title="Add startup logic to rcS">
    `rcS` runs after `rc.sysinit`. Place application-specific initialisation (network bring-up, sensor init, etc.) here.
  </Step>
</Steps>

## Login and authentication

NSH can require a password at login with `CONFIG_NSH_LOGIN=y`. Users are managed with the `useradd` and `userdel` built-in commands (require `CONFIG_NSH_LOGIN_PASSWD`).

```kconfig
CONFIG_NSH_LOGIN=y
CONFIG_NSH_LOGIN_PASSWD=y      # passwd file authentication
CONFIG_NSH_LOGIN_USERNAME="admin"
CONFIG_NSH_LOGIN_PASSWORD="secret"
```

## Key configuration options

| Option | Default | Description |
|--------|---------|-------------|
| `CONFIG_NSH_CONSOLE` | `y` | Enable serial console front-end |
| `CONFIG_NSH_TELNET` | `n` | Enable Telnet front-end |
| `CONFIG_NSH_READLINE` | `y` | Minimal readline (backspace only) |
| `CONFIG_NSH_CLE` | `n` | EMACS-like command line editor (+1.5 KB) |
| `CONFIG_NSH_LINELEN` | `80` | Max command line length |
| `CONFIG_NSH_MAXARGUMENTS` | `6` | Max arguments per command |
| `CONFIG_NSH_BUILTIN_APPS` | `y` | Allow registered built-in apps |
| `CONFIG_NSH_ROMFSETC` | `n` | Mount ROMFS /etc with startup scripts |
| `CONFIG_NSH_DISABLESCRIPT` | `n` | Disable scripting to save flash |
| `CONFIG_NSH_DISABLEBG` | `n` | Disable background (`&`) commands |
| `CONFIG_NSH_VARS` | `n` | Add local NSH variables (not env) |
| `CONFIG_NSH_FILEIOSIZE` | `1024` | Static I/O buffer size for file access |
| `CONFIG_NSH_TELNETD_PORT` | `23` | Telnet daemon listen port |
